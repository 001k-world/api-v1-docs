    Deposits

    Here you can get the details for refilling the cryptocurrency wallet, as well as the history of deposits.

    Generate Deposit Address

    Returns a new deposit address for a cryptocurrency according to the given parameters.

    POST /api/v1/deposit/generate_address

    Authentication: Required

    Permissions: AllowDeposits

    Parameters: request body

    Parameter Type Rest. Mandatory Description
    currency string max len 10 yes currency code
    method string max len 20 no method code, make a clarification if the currency is accepted by more than one method.
    if not specified, the first method in the list will be selected
    custom_id string max len 20 no user-specified unique id. unique.
    callback_url string max len 512 no URL for notification of updates

    Response:

    Parameter Type Description
    currency string specified currency code
    method string code of the specified or automatically selected method
    custom_id string user-specified unique id
    callback_url string if specified by the user, then the URL where updates on all deposits for this address will be sent
    address string deposit address in the selected network. determined by the method
    tag string can be empty if the selected method does not require an additional attribute

    Errors:

    • all common errors
    • type: permission_denied, code allow_deposits
    • validation error for each request parameter
    • address generation is temporarily unavailable for the selected method

    Get Deposit List

    Returns a list of deposits

    By default, it uses backward ordering.

    GET /api/v1/deposit

    Authentication: Required

    Permissions: AllowDeposits

    Parameters: query string

    Parameter Type Rest. Mandatory Description
    currency string max len 10 no currency code
    method string max len 20 no method code, make a clarification if the currency is accepted by more than one method.
    if not specified, the first method in the list will be selected
    custom_id string max len 20 no customer id for new address
    limit integer max 200 no limit on the number of records returned
    offset integer - no how many records to shift the selection

    Response: array of objects

    Parameter Type Description
    code string unique identifier of the deposit
    created_at datetime deposit creation time
    updated_at datetime last modified time
    closed_at datetime time of obtaining the final status. empty if not closed
    amount decimal the amount credited to the balance
    fee_amount decimal the amount of commissions withheld from the deposit. reference
    currency string deposit currency
    method string deposit method
    network string network where the deposit is accepted
    contract string contract data object if it is a token
    required_confirmations integer the number of confirmations in the blockchain required for a successful deposit
    actual_confirmations integer current number of confirmations
    status string current status of the deposit. all statuses: pending, success, failed
    reason_code string deposit rejection reason program code
    reason_message string detailed description of the reason for the rejection of the deposit
    txid string blockchain transaction ID
    attributes object attributes where the deposit received, if it is a cryptocurrency
    attributes.custom_id string user-specified identifier
    attributes.address string wallet address
    attributes.tag string additional tag if needed

    Note

    The same format of deposit details will be for callbacks
