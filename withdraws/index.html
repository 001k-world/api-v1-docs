    Withdraws

    Here are methods for working with withdrawals to external wallets.

    Create Withdraw Request

    Create a withdrawal request.

    POST /api/v1/withdraw

    Authentication: Required

    Permissions: AllowWithdraws

    Parameters: request body

    Parameter Type Rest. Mandatory Description
    currency string max len 10 yes currency code
    method string max len 20 no method code, make a clarification if the currency is accepted by more than one method.
    if not specified, the first method in the list will be selected
    amount decimal see method yes withdrawal amount
    receive_mode bool no default false. if true, then the amount parameter will indicate the amount that will be credited to address considering the commissions.
    address string see method yes Recipient address. There may also be a card or account number, depending on the chosen method.
    tag string see method cond Additional attribute for currencies where there is a tag or memo.
    custom_id string max len 20 no user-specified unique id. unique.
    callback_url string max len 512 no URL for notification of updates
    beneficiary object cond Usually for fiat currencies. An object with information about the recipient of the withdrawal.

    Response:

    Parameter Type Description
    code string Unique identifier of the withdrawal request
    created_at datetime Withdrawal creation time
    updated_at datetime Last modified time
    closed_at datetime Time of obtaining the final status. empty if not closed
    amount decimal Requested amount. to be deducted from account balance
    fee_amount decimal Amount of fees calculated form requested amount
    paid_amount decimal Amount actually deducted from the balance. Can be less than than requested amount in case of partial payment. Only for fiat currencies
    paid_fee_amount decimal Amount actually deducted from the balance as commissions
    currency string Withdrawal currency
    method string Withdrawal method
    network string Network where the withdrawal is created
    contract string Contract data object if it is a token
    status string Current status of the withdrawal request. all statuses: pending, success, failed
    reason_code string Withdrawal rejection reason program code
    reason_message string Detailed description of the reason for the rejection of the withdrawal request
    txid string Blockchain transaction ID
    custom_id string User-specified identifier
    address string Destination wallet address on account number
    tag string Additional tag if needed

    Errors:

    • all common errors
    • validation errors
    • out_of_balance with details

    Get Withdraw List

    Get a list of withdrawal requests.

    By default, it uses backward ordering. 

    GET /api/v1/withdraw

    Authentication: Required

    Permissions: AllowWithdraws

    Parameters: query string

    Parameter Type Rest. Mandatory Description
    currency string max len 10 no currency code
    method string max len 20 no method code
    custom_id string max len 20 no user-specified withdrawal id
    limit integer max 200 no limit on the number of records returned
    offset integer - no how many records to shift the selection

    Response: array of objects

    Errors:

    • all common errors
    • validation errors

    see same format as in request response

    Get Withdraw Details

    Get withdrawal request details by internal id.

    GET /api/v1/withdraw/:id

    Authentication: Required

    Permissions: AllowDeposits

    Parameters: path parameter

    Parameter Type Rest. Mandatory Description
    id string max len 20 yes unique withdraw code

    Response: see same format as in request response

    Errors:

    • all common errors
    • validation errors
    • not found error
