    Authentication

    Authentication for all private methods is done through request headers.

    There are three mandatory headers for authentication

    • X-Access-Key - API key identifier
    • X-Timestamp - UNIX Timestamp in milliseconds
    • X-Signature - cryptographic signature, HMAC SHA256

    API Key

    In order to get the access key and the secret you need to contact the administrator. Soon this will be an automated process.

    Timestamp

    This parameter is needed to protect against repeated requests with the same signature and not to execute an "outdated" request if the inter-service communication takes time. We check that the time on the client and on the server does not diverge too much. A difference of [-5, +5] seconds is allowed

    Signature

    To properly sign the request, you need to correctly prepare the request data. Let us have:

    access_key="<string>"
request_path="/api/v1/super_method"
timestamp=1673255381423
data="<string>" # - UTF-8 string with the body of the request. 
                # If you do not put anything in the body of the request,
                # then the empty string.

    To complete the signature, you need to combine the request data into one string as follows:

    payload=f"{access_key}{request_path}{timestap}{data}"

    Next, through HMAC and with your secret key, sign with the SHA256 hash algorithm, take the result in a hex-string in lower case. The received string must be set to the header X-Signature

    On the server, we will calculate the signature in the same order. And if it does not match, the request will not be processed.

    Signature and request implementation examples:

    # Python 3 Example
import hashlib
import hmac
import json
import time
import requests

# API domain and https schema 
BASE_URL = 'https://api.001k.world'

# Your credentials
access_key = "<>"
secret_key = "<>"

request_path = "/api/v1/super_method"

# request data
request_data = {
  "currency": "BTC",
  "amount": "17.3233",
}

# convert object into string
# use this string in request later 
data = json.dumps(request_data)

# current timestamp in milliseconds
timestamp = int(time.time() * 1000)

# prepare message for signing
message = f"{access_key}{request_path}{timestamp}{data}"
# calculate signature
signature = hmac.new(secret_key.encode('utf-8'), message.encode('utf-8'), hashlib.sha256).hexdigest()

# create url from base and path
url = f'{BASE_URL}{request_path}'

headers = {
    'Content-type': 'application/json',
    'X-Access-Key': access_key,
    'X-Timestamp': str(timestap),
    'X-Signature': signature,
}

# make request
response = requests.post(url=url, headers=headers, data=data)

# check the result
print(response.status_code)
print(response.content)

    // JavaScript, NodeJS example  
const axios = require('axios');
const crypto = require('crypto');

// API domain and https schema
const BASE_URL = 'https://api.001k.world';

// Your credentials
const access_key = '<>';
const secret_key = '<>';

const request_path = '/api/v1/super_method';

// request data
const request_data = {
  currency: 'BTC',
  amount: '17.3233',
};

// convert object into string
// use this string in request later
const data = JSON.stringify(request_data);

// current timestamp in milliseconds
const timestamp = Date.now();

// prepare message for signing
const message = `${access_key}${request_path}${timestamp}${data}`;

// calculate signature
const signature = crypto
  .createHmac('sha256', secret_key)
  .update(message)
  .digest('hex');

// create url from base and path
const url = `${BASE_URL}${request_path}`;

const headers = {
  'Content-type': 'application/json',
  'X-Access-Key': access_key,
  'X-Timestamp': `${timestamp}`,
  'X-Signature': signature,
};

// make request
axios
  .post(url, data, { headers })
  .then(response => {
    console.log(response);
  })
  .catch(err => {
    console.log(err?.response?.data || err);
  });

    Errors

    Errors that may occur during authentication.

    Code Description
    access_key.missed Header X-Access-Key missed
    access_key.invalid Access key has an invalid format or not found in the database
    access_key.inactive Access key set inactive
    access_key.ip_whitelist Client IP address check failed
    user.inactive User is deactivated or blocked
    timestamp.missed Header X-Timestamp missed
    timestamp.invalid Header X-Timestamp has invalid format or out of bounds [-5, +5] seconds
    signature.missed Header X-Signature missed
    signature.invalid Signature mismatch

    These errors can occur in any method that requires authentication.
